Grundlagen der OOP

Was ist eine Klasse?

Eine Klasse ist ein Bauplan oder eine Vorlage. Sie beschreibt, welche Eigenschaften (Attribute) und Fähigkeiten (Methoden) etwas hat – aber sie ist noch kein konkretes Ding.

Analogie: Eine Klasse ist wie ein Kuchenrezept. Das Rezept beschreibt, wie der Kuchen aussieht – aber es ist noch kein echter Kuchen. Erst wenn du nach dem Rezept backst, entsteht ein Objekt (= echter Kuchen).

Was ist ein Objekt?

Ein Objekt ist ein konkretes Exemplar einer Klasse. Es wurde nach dem Bauplan (der Klasse) erzeugt und hat echte Werte für seine Eigenschaften.

Klasse

  • Bauplan / Vorlage
  • Beschreibt was ein Ding ist
  • Existiert nur einmal im Code
  • Hat noch keine konkreten Werte

Objekt (= Instanz)

  • Konkretes Exemplar der Klasse
  • Hat echte, individuelle Werte
  • Kann viele Male erzeugt werden
  • Verhält sich laut Bauplan

Instanz & Abstraktion einfach erklärt

Instanz

Instanz = anderes Wort für Objekt. Wenn du ein Objekt aus einer Klasse erzeugst, nennt man das „eine Instanz der Klasse erstellen".

bello ist eine Instanz der Klasse Hund.

Abstraktion

Abstraktion bedeutet: Du legst nur das Wesentliche fest und lässt unnötige Details weg. Eine Klasse Auto hat z. B. farbe und marke – aber keine Reifenprofiltiefe (wäre zu viel Detail).

UML & Diagramme

UML steht für Unified Modeling Language – eine Standardsprache, um Programme grafisch darzustellen, bevor man sie codiert.

Klassendiagramm

Ein Klassendiagramm zeigt den Bauplan einer Klasse: Name, Attribute (mit Datentyp) und Methoden (mit Parameter & Rückgabetyp).

Hund
− name : str
− alter : int
− rasse : str
+ __init__(name, alter, rasse)
+ bellen() : str
+ get_name() : str
Klassendiagramm (Bauplan)
Legende der Symbole:
+ = public (öffentlich)  |  − = private (privat)  |  # = protected (geschützt)

Objektdiagramm

Ein Objektdiagramm zeigt konkrete Objekte mit ihren echten Werten – also einen Schnappschuss des Programms zu einem bestimmten Zeitpunkt. Der Name ist unterstrichen und besteht aus: objektname : Klassenname

bello : Hund
name = "Bello"
alter = 3
rasse = "Labrador"
Objekt 1
luna : Hund
name = "Luna"
alter = 5
rasse = "Husky"
Objekt 2

Vom Klassendiagramm zum Objektdiagramm

So geht's Schritt für Schritt:

  1. Gib dem Objekt einen Namen (z. B. bello) und schreibe: bello : Hund
  2. Übernehme alle Attribute aus dem Klassendiagramm
  3. Ersetze die Datentypen durch echte Werte (str → "Bello")
  4. Methoden weglassen – sie kommen von der Klasse

Implementationsdiagramm (kurz erklärt)

Ein Implementationsdiagramm zeigt, wie der Code wirklich aufgebaut ist – also welche Dateien/Module existieren und wie sie zusammenarbeiten. Es ist weniger abstrakt als ein Klassendiagramm und näher am echten Code.

Im Unterricht ist das Klassendiagramm am wichtigsten. Das Implementationsdiagramm ist eher für größere Projekte relevant.

Beziehungen zwischen Klassen

Assoziation

Eine Assoziation bedeutet: Zwei Klassen kennen sich und können miteinander kommunizieren – aber keine Klasse besitzt die andere. Sie sind unabhängig voneinander.

Beispiel: Eine Person fährt ein Auto. Das Auto existiert auch ohne die Person.

Person
name : str
───────── fährt ─────────→
Assoziation
Auto
marke : str
python · assoziation.py
class Auto:
    def __init__(self, marke: str) -> None:
        self.marke = marke  # Attribut

class Person:
    def __init__(self, name: str) -> None:
        self.name = name

    def fahren(self, auto: Auto) -> str:
        return f"{self.name} fährt {auto.marke}"  # Assoziation: Person kennt Auto

# Objekte erzeugen
mein_auto = Auto("BMW")
anna = Person("Anna")
print(anna.fahren(mein_auto))  # → "Anna fährt BMW"

Aggregation

Eine Aggregation ist eine stärkere Beziehung: Die eine Klasse enthält die andere – aber die enthaltene Klasse kann auch alleine existieren. Im Diagramm erkennst du sie an einer offenen Raute (◇).

Merkhilfe: Bei Aggregation gilt: „Das Teil kann ohne das Ganze leben."
Ein Schüler (Teil) existiert auch, wenn seine Klasse (Ganze) aufgelöst wird.

Beispiel: Eine Schulklasse enthält mehrere Schueler. Die Schüler existieren aber auch, wenn die Klasse aufgelöst wird.

Schulklasse
name : str
◇────────────
Aggregation  1      0..*
Schueler
name : str

Multiplizität: 1 Schulklasse hat 0..* (null bis viele) Schüler

python · aggregation.py
class Schueler:
    def __init__(self, name: str) -> None:
        self.name = name

class Schulklasse:
    def __init__(self, bezeichnung: str) -> None:
        self.bezeichnung = bezeichnung
        self.schueler: list[Schueler] = []  # Liste enthält Schueler-Objekte

    def add_schueler(self, s: Schueler) -> None:
        self.schueler.append(s)  # Schüler zur Klasse hinzufügen

    def zeige_liste(self) -> None:
        for s in self.schueler:
            print(s.name)

# Schüler können unabhängig von der Klasse existieren
tim = Schueler("Tim")
lea = Schueler("Lea")
klasse_10a = Schulklasse("10a")
klasse_10a.add_schueler(tim)
klasse_10a.add_schueler(lea)
klasse_10a.zeige_liste()  # → Tim, Lea

Sichtbarkeiten

Sichtbarkeit bestimmt, wer auf ein Attribut oder eine Methode zugreifen darf.

SymbolNamePython-SchreibweiseZugriff
+ public self.name Von überall erreichbar
# protected self._name (ein Unterstrich) Nur in der Klasse & Unterklassen (Konvention)
− private self.__name (zwei Unterstriche) Nur innerhalb der Klasse selbst
In Python sind Sichtbarkeiten teilweise Konvention, kein hartes Verbot. Zwei Unterstriche (__) erzeugen aber echtes Name-Mangling und erschweren externen Zugriff wirklich.
python · sichtbarkeit.py
class Bankkonto:
    def __init__(self, inhaber: str, kontostand: float) -> None:
        self.inhaber = inhaber          # public  → jeder darf zugreifen
        self._bank = "Sparkasse"       # protected → nur intern & Unterklassen
        self.__kontostand = kontostand  # private → nur innerhalb der Klasse

    def get_kontostand(self) -> float:  # Getter
        return self.__kontostand

    def set_kontostand(self, betrag: float) -> None:  # Setter
        if betrag >= 0:  # Validierung!
            self.__kontostand = betrag

konto = Bankkonto("Max", 1000.0)
print(konto.inhaber)           # ✅ "Max"
print(konto.get_kontostand())  # ✅ 1000.0 (über Getter)
# print(konto.__kontostand)   # ❌ Fehler! Direkt nicht erlaubt

Warum Getter & Setter?

Problem ohne Getter/Setter

Wenn Attribute public sind, kann jeder beliebige (auch ungültige) Werte setzen:

konto.kontostand = -99999 # 😱 Möglich!

Lösung mit Getter/Setter

Ein Setter kann Werte prüfen, bevor sie gespeichert werden – Datenkapselung:

konto.set_kontostand(-1) # 🛡️ Wird ignoriert

Programmierung in Python

Klasse definieren (mit __init__)

__init__ ist der Konstruktor – er wird automatisch aufgerufen, wenn ein Objekt erzeugt wird.

python · klasse_definieren.py
class Tier:
    """Eine einfache Tier-Klasse als Beispiel."""

    def __init__(self, name: str, alter: int, tierart: str) -> None:
        # self = Verweis auf das aktuelle Objekt
        self.name = name          # Attribut speichern
        self.alter = alter
        self.tierart = tierart

    def beschreibung(self) -> str:
        # Methode gibt einen String zurück
        return f"{self.name} ist ein {self.tierart} und {self.alter} Jahre alt."

    def geburtstag(self) -> None:
        # Methode verändert ein Attribut
        self.alter += 1
        print(f"{self.name} ist jetzt {self.alter}!")

Objekte erzeugen und verwenden

python · objekte.py
# Objekte erzeugen (= Instanzen erstellen)
katze1 = Tier("Mia", 3, "Katze")   # __init__ wird aufgerufen
katze2 = Tier("Leo", 7, "Kater")

# Auf Attribute zugreifen
print(katze1.name)   # → "Mia"
print(katze2.alter)  # → 7

# Methoden aufrufen
print(katze1.beschreibung())  # → "Mia ist eine Katze und 3 Jahre alt."
katze1.geburtstag()           # → "Mia ist jetzt 4!"

# Attribut direkt ändern (wenn public)
katze2.name = "Simba"
print(katze2.name)  # → "Simba"

Type Hints – Parameter und Rückgabewerte

Type Hints machen den Code lesbarer und helfen der IDE bei der Fehlersuche. Sie sind optional, aber empfohlen.

python · type_hints.py
class Rechner:

    # Parameter: a und b sind int, Rückgabe ist int
    def addiere(self, a: int, b: int) -> int:
        return a + b

    # Rückgabe ist float
    def teile(self, a: float, b: float) -> float:
        return a / b

    # Kein Rückgabewert → None
    def zeige_info(self, text: str) -> None:
        print(f"Info: {text}")

    # Parameter kann None sein → Optional mit | None
    def set_name(self, name: str | None) -> None:
        self.name = name

Vollständiges Beispiel: Klasse mit allem

python · vollstaendiges_beispiel.py
class Schueler:
    """Repräsentiert einen Schüler mit Name, Alter und Noten."""

    def __init__(self, name: str, alter: int) -> None:
        self.name = name
        self.alter = alter
        self.__noten: list[int] = []  # private: Noten geschützt

    def note_hinzufuegen(self, note: int) -> None:
        if 1 <= note <= 6:  # Validierung
            self.__noten.append(note)
        else:
            print("Ungültige Note!")

    def get_durchschnitt(self) -> float:
        if not self.__noten:
            return 0.0
        return sum(self.__noten) / len(self.__noten)

    def get_noten(self) -> list[int]:  # Getter
        return self.__noten.copy()  # Kopie zurückgeben

    def __str__(self) -> str:  # Für print()-Ausgabe
        return f"{self.name} ({self.alter} J.) – Ø {self.get_durchschnitt():.1f}"


# --- Hauptprogramm ---
klasse: list[Schueler] = []

s1 = Schueler("Anna", 16)
s1.note_hinzufuegen(2)
s1.note_hinzufuegen(1)
s1.note_hinzufuegen(3)

s2 = Schueler("Ben", 17)
s2.note_hinzufuegen(4)
s2.note_hinzufuegen(3)

klasse.extend([s1, s2])

for schueler in klasse:
    print(schueler)  # nutzt __str__ automatisch
# → Anna (16 J.) – Ø 2.0
# → Ben  (17 J.) – Ø 3.5

Namenskonventionen in Python

Python hat offizielle Regeln für Bezeichnungen (definiert im PEP 8 Style Guide):

Was?StilBeispieleFalsch
Klassen PascalCase Schueler, BankKonto, MeinTier schueler, bank_konto
Methoden snake_case get_name(), berechne_sum(), zeige_info() GetName(), berechneSumme()
Attribute / Variablen snake_case mein_name, alter, kontostand MeinName, meinName
Konstanten UPPER_CASE MAX_GROESSE, PI, STANDARD_WERT max_groesse, pi
Private Attribute __snake_case __kontostand, __passwort kontostand_ (ein Unterstrich allein)
Protected Attribute _snake_case _intern, _bank_name
Dateien/Module snake_case mein_programm.py, bank_konto.py MeinProgramm.py
python · namenskonventionen.py
# ✅ Korrekte Namensgebung in Python

MAX_PUNKTE = 100  # Konstante → UPPER_CASE

class SpielCharakter:  # Klasse → PascalCase

    def __init__(self, spieler_name: str, lebenspunkte: int) -> None:
        self.spieler_name = spieler_name   # Attribut → snake_case
        self.__lebenspunkte = lebenspunkte  # privat → __snake_case
        self._level = 1                    # protected → _snake_case

    def get_lebenspunkte(self) -> int:   # Methode → snake_case
        return self.__lebenspunkte

    def erhoehe_level(self) -> None:    # Methode → snake_case
        self._level += 1

held = SpielCharakter("Aragorn", 100)  # Variable → snake_case
print(held.spieler_name)               # → "Aragorn"
PascalCase = Jedes Wort beginnt mit Großbuchstabe: MeinKlassenName
snake_case = Alles klein, Wörter mit Unterstrich: mein_methoden_name
UPPER_CASE = Alles groß, Wörter mit Unterstrich: MEINE_KONSTANTE
